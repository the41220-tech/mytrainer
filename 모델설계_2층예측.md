# 예측 모델 설계 — 2층(개인 회귀 × 경력 프라이어) **v2: 둔화 반영**

> 위치: `src/engine.ts` (`blendedWeeklyGain`, `goalETA`, `etaWithDecay`). 결정적·실행검증(6/6).
> 목적: 한 사람의 **작은 데이터**로 성장을 **과장 없이** 예측(과적합·낙관 외삽 방지).
> v1 문제(피드백 반영): 직선 외삽 + 프라이어 과대 → "벤치 93→100을 3주"처럼 **너무 진보적**. v2에서 교정.

## 왜 '큰 ML'이 아니라 이 모델인가
한 사람의 근력 기록은 수십~수백 행. 신경망/대형 모델은 과적합. 그래서 **개인 회귀 + 모집단 프라이어를 섞는 empirical-Bayes shrinkage**가 정답 — 데이터가 적을수록 일반 통념(프라이어)에 기대고, 쌓일수록 '나'에 수렴.

## 2층 구조 (v2)
1. **개인 층(personal fit)**: 종목별 e1RM 시계열의 **최근 6세션** 선형회귀 → 현재 주간 slope(kg/주). (전체기간이 아닌 **최근**을 써서 초반 빠른 구간의 과대평가 제거 = 둔화 자연 반영)
2. **프라이어 층(population prior)**: 경력별 복합운동 기대 주간 증가 — **하향 보정**(실측 근사)
   - 초급 **1.5** · 중급 **0.5** · 고급 **0.2** kg/주 (v1은 2.5/1.0/0.3 → 과대였음)

**혼합**: `w = n / (n+8)` (강한 shrinkage) → `gain = min(w·개인 + (1-w)·프라이어, 프라이어×3)`
- n=2 → w≈0.20(프라이어 위주), n=8 → 0.50, n=16 → 0.67(개인 위주)
- **상한(cap = 프라이어×3)**: 단기 급상승을 그대로 외삽하지 않음.
- **정체 가드**: n≥6 이고 개인 slope ≤ 0이면 프라이어로 가리지 않고 **정체로 처리**(예측 거부 → 디로드 권고).

## 둔화(체감) ETA — v2 핵심
직선(`gap / gain`)이 아니라 **매주 증가율이 프라이어 쪽으로 감쇠**(`g ← prior + (g−prior)·0.85`)하며 누적 → 목표가 멀수록 **비선형으로 느려짐**(실제 근력 곡선). 출력은 단일값이 아닌 **범위**:
- **낙관**(현재 속도 유지) ≤ **중심**(둔화 반영) ≤ **보수**(경력 평균까지 둔화) = `gap/prior`.

## 사용처
`predict_goal`이 프로필 경력으로 혼합 증가율을 구해 **목표 1RM 도달 ETA(중심 + 범위)** 계산. 출력에 **모델 구성(개인 X% × 경력 Y%·상한)**을 표기 → 투명성.

## 검증 (실행, 시드 벤치 93.3→100kg)
| 입력 | 결과 |
|---|---|
| 최근 slope 2.9 · n8 · 중급 | **주간 +1.5kg**(상한 적용) → **중심 6주**, 범위 4~13주 |
| 동일 데이터 · 초급 | 중심 4주(주간 +2.2) — 빠름 |
| 동일 데이터 · 고급 | 중심 21주(주간 +0.6) — 느림 |
| slope 0 · n8 | 0 → 정체(예측 거부) |
| **v1 대비** | v1 ~3주(주간 2.3) → **v2 ~6주** (현실적으로 둔화) |

## 부상 예측 모델
부상 데이터는 한 사람당 표본이 극소 → **ML 학습 불가**. 대신 **ACWR(급성:만성 작업부하비)** 개인화 조기경보(>1.5 위험) 휴리스틱 채택. (`injury_risk`)
